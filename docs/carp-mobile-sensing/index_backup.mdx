The CARP Mobile Sensing (CAMS) Flutter package [carp_mobile_sensing](https://pub.dartlang.org/packages/carp_mobile_sensing) is a programming framework for adding digital phenotyping capabilities to your mobile (health) app.

## Purpose and Goals

The overall goal of CAMS is to have a programming framework that helps build custom mobile sensing applications.

The basic scenario we want to support is to allow programmers to design and implement a custom mHealth app, e.g., for [cardiovascular diseases](https://carp.dk/mcardia/) or [diabetes](https://carp.dk/diafocus/). Such an app would have its main focus on providing application-specific functionality for patients with either heart rhythm problems or diabetes, which are two rather distinct application domains. 

However, the goal of a mobile sensing programming framework would be to enable programmers to add mobile sensing capabilities in a 'flexible and simple' manner. This would include adding support for collecting data like ECG, location, activity, and step counts; to format this data according to different health data formats (like the [Open mHealth](https://www.openmhealth.org) formats); to use this data in the app (e.g. showing it to the user); and to upload it to a specific server, using a specific API (e.g. REST), in a specific format. 
The framework should be able to support different wearable devices for ECG  or glucose monitoring. Hence, the focus is on software engineering support in terms of a sound programming API and runtime execution environment, which is being maintained as the underlying mobile phone operating systems are evolving. Moreover, focus is on providing an extensible API and runtime environment, which allows for adding application-specific data sampling, wearable devices, data formatting, data management, and data uploading functionality.

## Features

<Tabs>
  <Tab title="Reading Data">
    <Icon icon="book-open" size={32} />
    
    **Flexible Data Queries**
    - Read individual data points
    - Get time-bucketed aggregates (hourly, daily, weekly)
    - Fetch workout summaries
    - Calculate step totals
    
    ```dart
    // Get last 24 hours of step data
    final data = await health.getHealthDataFromTypes(
      types: [HealthDataType.STEPS],
      startDate: yesterday,
      endDate: now,
    );
    ```
  </Tab>
  
  <Tab title="Writing Data">
    <Icon icon="pen-to-square" size={32} />
    
    **Easy Data Recording**
    - Write numeric health data
    - Save workout sessions
    - Log meals with nutrition info
    - Record specialized data (ECG, audiograms, etc.)
    
    ```dart
    // Save weight measurement
    await health.writeHealthData(
      value: 70.5,
      type: HealthDataType.WEIGHT,
      startTime: DateTime.now(),
    );
    ```
  </Tab>
  
  <Tab title="Permissions">
    <Icon icon="shield-halved" size={32} />
    
    **Permission Management**
    - Request granular permissions
    - Check permission status
    - Handle read/write access separately
    - Platform-specific permission flows
    
    ```dart
    // Request read access
    final granted = await health.requestAuthorization(
      [HealthDataType.STEPS],
      permissions: [HealthDataAccess.READ],
    );
    ```
  </Tab>
</Tabs>

## Quick Example

Here's a complete example to get you started:

```dart
import 'package:health/health.dart';

// 1. Create Health instance
final health = Health();
await health.configure();

// 2. Request permissions
final types = [HealthDataType.STEPS];
final granted = await health.requestAuthorization(types);

if (granted) {
  // 3. Read today's steps
  final now = DateTime.now();
  final midnight = DateTime(now.year, now.month, now.day);
  final steps = await health.getTotalStepsInInterval(midnight, now);
  
  print('Steps today: ${steps ?? 0}');
}
```
