This page contains an overview of the measure types available in sampling packages; both packages that are built-in to the [carp_mobile_sensing](https://pub.dev/packages/carp_mobile_sensing) framework, as well as external ones.
This list is constantly evolving as support is expanded.

## Measure Type Format

The [`dataType`](https://pub.dev/documentation/carp_core/latest/carp_core_common/Measure/dataType.html) of a [`Measure`](https://pub.dev/documentation/carp_core/latest/carp_core_common/Measure-class.html) specifies what type of data to collect.
This string version of [`DataType`](https://pub.dev/documentation/carp_core/latest/carp_core_common/DataType-class.html) is typically on the namespace + name format, where namespace is usually `dk.cachet.carp` and name is the measure `type`:

```text
dk.cachet.carp.<type>
```

The `type` is specified in the tables below.

## Event-based vs. One-Time Measures

Most measures are **event-based** (EB), i.e., once triggered they continuously sample data from their sensor.
Some measures are **one-time** (OT), i.e., they sample one piece of data when triggered.

Knowing EB vs. OT matters because event-based measures can run until stopped, while one-time measures need to be triggered explicitly (for example using a [`PeriodicTrigger`](https://pub.dev/documentation/carp_mobile_sensing/latest/domain/PeriodicTrigger-class.html)).

The difference in protocol setup is illustrated below:

```dart
// Collect device info only once, when this study is deployed.
protocol.addTaskControl(
	OneTimeTrigger(),
	BackgroundTask(
			measures: [Measure(type: DeviceSamplingPackage.DEVICE_INFORMATION)]),
	phone,
);

// Immediately start collecting event-based measures.
protocol.addTaskControl(
	ImmediateTrigger(),
	BackgroundTask(measures: [
		Measure(type: SensorSamplingPackage.STEP_EVENT),
		Measure(type: SensorSamplingPackage.AMBIENT_LIGHT),
		Measure(type: DeviceSamplingPackage.SCREEN_EVENT),
		Measure(type: DeviceSamplingPackage.BATTERY_STATE),
	]),
	phone,
);

// Collect weather (one-time measure) every 5 minutes.
protocol.addTaskControl(
		PeriodicTrigger(period: Duration(minutes: 5)),
		BackgroundTask(measures: [Measure(type: ContextSamplingPackage.WEATHER)]),
		weatherService);
```

Each [`SamplingPackage`](https://pub.dev/documentation/carp_mobile_sensing/latest/runtime/SamplingPackage-class.html) provides supported types through its [`dataTypes`](https://pub.dartlang.org/documentation/carp_mobile_sensing/latest/runtime/SamplingPackage/dataTypes.html) property, and each package also provides default [sampling schemes](https://pub.dev/documentation/carp_mobile_sensing/latest/runtime/SamplingPackage/samplingSchemes.html).

## Available Measure Types

The tables below describe, for each measure:

- **Type Name** (`dk.cachet.carp.<type>`)
- **Availability** on Android and/or iOS
- **Sampling package** containing the measure
- Measure type (**event-based** or **one-time**)
- Data class collected (for built-in measures)
- A short description
